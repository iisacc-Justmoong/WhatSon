# 문서 로드맵 (Draft)

## 1차(현재)

- 완료: 디렉터리 골격 구성
- 완료: 구현 중심 초안 문서 작성

## 2차(다음)

- 아키텍처 다이어그램(부트스트랩, 데이터 흐름, 계층 상호작용)
- 도메인별 파일 포맷 명세서
- 문제해결 가이드(권한, 경로, 플랫폼별 빌드 실패)

## 3차(안정화)

- 릴리스 노트 규격화
- 기여 규칙/코드 리뷰 체크리스트 문서화
- 운영 점검표(healthcheck/로그/복구 절차)

## 원칙

본 문서는 제품 구현의 단일 진실원천(Single Source of Truth)을 지향하되, 코드 변경보다 뒤처지지 않도록 작은 단위로 자주 갱신한다.
