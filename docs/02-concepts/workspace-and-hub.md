# Workspace와 Hub 개념 (Draft)

## 핵심 단위

- `*.wshub`: 워크스페이스 패키지 루트
- `*.wscontents`: 계층/보조 도메인 파일 묶음
- `Library.wslibrary`: 노트 저장/인덱스 중심 도메인
- `*.wsresources`: 첨부 리소스 도메인
- `*.wsstat`: 허브 통계/메타 도메인

## 런타임 해석 원칙

1. 앱은 blueprint 경로에서 첫 `.wshub`를 탐지한다.
2. 도메인별 파서/스토어/크리에이터로 분리된 계층에서 데이터가 정규화된다.
3. 병렬 로더가 도메인 스냅샷을 생성하고 메인 스레드 뷰모델에 적용한다.
4. 일부 도메인 실패가 전체 중단으로 번지지 않도록 도메인 단위 탄력성을 유지한다.

## 데이터 형식 철학

- 파서는 변형 스키마를 관대하게 수용한다(호환성 우선).
- 저장은 크리에이터가 표준 형태로 재생성한다(정규화 우선).
- 결과적으로 입력 유연성과 출력 일관성을 분리한다.
